# Transactions Report

The Transactions Report contains all transaction data including status, currency conversion, amounts, and customer details. Use this report to search transactions, issue refunds, void payments, and more.

<Frame caption="Transactions Report with filters and action buttons">
  ![Transactions Report](/images/control-panel/transactions-report.png)
</Frame>

---

## Accessing the Report

Navigate to **Operations > Transactions** in the Control Panel menu.

---

## Generating a Report

<Steps>
  <Step title="Set Date Range">
    Select the date range using the dropdown or custom date picker
  </Step>
  <Step title="Configure Filters">
    Add filters to narrow results (optional)
  </Step>
  <Step title="Run Report">
    Click **Run Report** to generate results
  </Step>
</Steps>

## Linkage Column

Each transaction row has two icons in the Linkage column:

| Icon | Description |
|------|-------------|
| **Lifecycle** | View the complete transaction lifecycle (auth, capture, refund, etc.) |
| **Details** | Open transaction details summary in a side panel |

## Available Actions

Right-click on a transaction or use the **Actions** button to access:

### Refund

Issue a refund for settled transactions.

| Field | Description |
|-------|-------------|
| Amount | Refund amount (up to original transaction amount) |
| Reason | Optional reason for the refund |

<Note>
Refunds can only be issued **after settlement** (typically 24+ hours after the transaction).
</Note>

### Instant Refund

Fast refund using Mastercard Send or Visa Direct.

| Feature | Description |
|---------|-------------|
| Speed | Funds delivered within **30 minutes** |
| Networks | Mastercard Send, Visa Direct |
| Availability | Requires enablement by Nuvei |

### Payout

Credit the customer **beyond** the original transaction amount.

<Warning>
Payouts require specific configuration and compliance approval.
</Warning>

### Void

Cancel a transaction **before settlement** (within 24 hours).

| Field | Description |
|-------|-------------|
| Reason | Optional reason for voiding |

<Tip>
Use void instead of refund when possible to avoid processing fees.
</Tip>

### Settle

Capture an authorized transaction (up to 7 days after authorization).

| Field | Description |
|-------|-------------|
| Amount | Capture amount (up to authorized amount) |

### Void Credit

Cancel a credit/refund before funds are returned to the customer.

### Fast Payout

Instant payout using Visa Direct or Mastercard MoneySend.

| Feature | Description |
|---------|-------------|
| Speed | Funds delivered within **30 minutes** |
| Use Case | Instant winnings, rapid refunds |

### Proof of Payment

Generate a PDF proof of payment (available for PIX APM transactions).

## Bulk Actions

Perform actions on multiple transactions simultaneously:

<Steps>
  <Step title="Select Transactions">
    Check the boxes next to transactions you want to process
  </Step>
  <Step title="Choose Action">
    Select the bulk action from the dropdown
  </Step>
  <Step title="Confirm">
    Review and confirm the bulk operation
  </Step>
</Steps>

<Note>
Bulk actions are limited to **20 transactions** per operation.
</Note>

## Report Parameters

### Transaction Tab

| Parameter | Description |
|-----------|-------------|
| Date | Transaction date and time |
| Transaction ID | Nuvei's unique transaction identifier |
| Related Transaction ID | ID of related transactions (refund to original, etc.) |
| Transaction Type | Sale, Auth, Settle, Credit, Refund, Void, Payout |
| Authorization Type | Final, Pre-Auth |
| Currency | Transaction currency |
| Amount | Transaction amount |
| Transaction Result | APPROVED, DECLINED, ERROR, PENDING |
| Reason Code | Decline/error reason code |
| Filter Reason | Fraud filter that triggered (if applicable) |
| Processing Channel | E-commerce, MOTO, POS |
| Acquiring Bank | Bank that processed the transaction |
| Multi-client | Parent merchant (if applicable) |
| Client Name | Merchant account name |
| Site Name | Merchant site name |

| Client Request ID | Your unique transaction identifier |
| Client Unique ID | Your customer identifier |
| Merchant Advice Code | Issuer advice for recurring |
| Authorization Code | Issuer authorization code |
| ARN | Acquirer Reference Number |

## Common Use Cases

<AccordionGroup>
  <Accordion title="Find a specific transaction">
    1. Go to Operations > Transactions
    2. Enter the **Transaction ID** or **Client Unique ID** in the filter
    3. Click Run Report
  </Accordion>
  
  <Accordion title="Issue a partial refund">
    1. Find the transaction
    2. Click Actions > Refund
    3. Enter the partial amount
    4. Confirm the refund
  </Accordion>
  
  <Accordion title="Check why a transaction was declined">
    1. Find the declined transaction
    2. Check the **Reason Code** column
    3. Review the **Authentication Tab** for 3DS issues
    4. Check the **Risk Tab** for fraud rule triggers
  </Accordion>
  
  <Accordion title="View transaction lifecycle">
    1. Find the transaction
    2. Click the **Lifecycle** icon in the Linkage column
    3. View all related transactions (auth, settle, refund)
  </Accordion>
</AccordionGroup>
