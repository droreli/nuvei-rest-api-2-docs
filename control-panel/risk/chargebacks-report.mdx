# Chargebacks Report

The Chargebacks Report displays all chargebacks filed against your transactions. Use this report to analyze dispute patterns, submit evidence, and track dispute outcomes.

<Frame caption="Chargebacks Report with filters and actions">
  ![Chargebacks Report](/images/control-panel/chargebacks-report.png)
</Frame>

---

## Accessing the Report

Navigate to **Risk > Chargebacks** in the Control Panel.

<Note>
The Chargebacks Report updates daily and displays data for the past year.
</Note>

---

## Generating a Report

<Steps>
  <Step title="Select Date Range">
    Choose the chargeback date range
  </Step>
  <Step title="Apply Filters">
    Add filters to narrow results (optional)
  </Step>
  <Step title="Run Report">
    Click **Run Report** to generate results
  </Step>
</Steps>

## Chargeback Types

| Type | Description |
|------|-------------|
| **Retrieval** | Information request (not yet a chargeback) |
| **Chargeback** | Formal dispute initiated by cardholder |
| **Pre-Arbitration** | Second chargeback after representment |
| **Arbitration** | Card network arbitration |

## Available Actions

### Contact Support

Request assistance from Nuvei support for complex cases.

### Refund

For **Retrieval** type chargebacks, you can issue a refund to prevent escalation:

<Steps>
  <Step title="Select Retrieval">
    Find the retrieval request
  </Step>
  <Step title="Click Refund">
    Select Refund action
  </Step>
  <Step title="Confirm Amount">
    Verify refund amount
  </Step>
  <Step title="Process">
    Submit the refund
  </Step>
</Steps>

<Tip>
Refunding a retrieval request often prevents it from becoming a full chargeback.
</Tip>

### Dispute Request

Submit evidence to dispute a chargeback:

<Steps>
  <Step title="Select Chargebacks">
    Check boxes next to chargebacks to dispute (up to 20)
  </Step>
  <Step title="Click Dispute Request">
    Click the Dispute Request button
  </Step>
  <Step title="Upload Documents">
    Upload supporting evidence (max 10 files)
  </Step>
  <Step title="Add Notes">
    Provide dispute explanation
  </Step>
  <Step title="Submit">
    Submit the dispute
  </Step>
</Steps>

#### Document Requirements

| Format | Max Size | Max Files |
|--------|----------|-----------|
| PDF | 4 MB | 10 |
| DOC/DOCX | 4 MB | 10 |
| TXT | 4 MB | 10 |

## Compelling Evidence

Submit appropriate evidence based on chargeback reason:

### Fraud Chargebacks (10.x, 4837, 4863)

| Evidence | Description |
|----------|-------------|
| 3DS authentication proof | ECI, CAVV, dsTransID |
| Device fingerprint | Matching device from previous purchases |
| Delivery confirmation | Signed delivery receipt |
| Customer communication | Emails confirming order |
| IP address match | Same IP as previous legitimate orders |

### Service/Merchandise Not Received (13.1, 4855)

| Evidence | Description |
|----------|-------------|
| Delivery confirmation | Tracking number, signature |
| Proof of delivery | Photo, GPS coordinates |
| Service completion | Screenshots, logs |
| Customer acknowledgment | Confirmation email |

### Cancelled Recurring (13.2, 4841)

| Evidence | Description |
|----------|-------------|
| Terms of service | Signed/agreed terms |
| Cancellation policy | Clear policy shown at signup |
| Communication | No cancellation request received |
| Billing notifications | Advance billing notices sent |

### Credit Not Processed (13.6, 4860)

| Evidence | Description |
|----------|-------------|
| Refund confirmation | Proof refund was issued |
| Transaction ID | Refund transaction reference |
| Bank statement | Evidence of credit |

## Chargeback Prevention

### Before the Chargeback

| Action | Description |
|--------|-------------|
| Clear billing descriptor | Customer recognizes charge |
| Confirmation emails | Order and shipping confirmations |
| 3DS authentication | Liability shift for fraud |
| Good customer service | Resolve issues before disputes |

### Use Pre-Chargeback Tools

| Tool | Benefit |
|------|---------|
| Ethoca Alerts | Early warning, auto-refund |
| Visa VRDR | Auto-accept eligible disputes |
| Verifi CDRN | Consumer dispute resolution |

## Best Practices

<AccordionGroup>
  <Accordion title="Respond within deadlines">
    - **Visa**: 30 days from chargeback date
    - **Mastercard**: 45 days from chargeback date
    - Missing deadlines = automatic loss
  </Accordion>
  
  <Accordion title="Keep detailed records">
    Maintain for at least 18 months:
    - Transaction details
    - Customer communications
    - Delivery confirmations
    - 3DS authentication data
  </Accordion>
  
  <Accordion title="Analyze patterns">
    Review chargebacks for patterns:
    - Specific products
    - Geographic regions
    - Customer segments
    - Time periods
  </Accordion>
  
  <Accordion title="Use 3DS consistently">
    3DS provides liability shift for fraud chargebacks when authentication is successful.
  </Accordion>
</AccordionGroup>
