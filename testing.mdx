# Testing

Use the sandbox environment and test cards to validate your integration.

## Sandbox Environment

| Setting | Value |
|---------|-------|
| Base URL | `https://api-sandbox.nuvei.com/payment-api` |
| API Key | Use your sandbox API key |
| Real charges | No |

## Test Card Details

For all test cards, use:

| Field | Value |
|-------|-------|
| Expiration Month | Any future month (`12`) |
| Expiration Year | Any future year (`2030`) |
| CVV | Any 3 digits (`217`) |
| Cardholder Name | Any name (`John Smith`) |

## Test Amounts

Use specific amounts to trigger behaviors:

| Amount | Result |
|--------|--------|
| Any | Approved (with approved card) |
| `666` | Declined |
| `777` | Error response |

## Sample Test Flow

<Steps>
  <Step title="Create a Sale payment">
```bash
curl -X POST https://api-sandbox.nuvei.com/payment-api/payments \
  -H "Content-Type: application/json" \
  -H "x-api-key: YOUR_SANDBOX_KEY" \
  -d '{
    "processingEntityId": "1234567890",
    "transactionType": "Sale",
    "amount": 100,
    "currency": "USD",
    "paymentMethod": {
      "type": "card",
      "card": {
        "cardNumber": "4000027891380961",
        "expirationMonth": "12",
        "expirationYear": "2030",
        "cvv": "217"
      }
    }
  }'
```
  </Step>
  
  <Step title="Verify response">
```json
{
  "paymentId": "375011",
  "result": {
    "status": "approved"
  },
  "authCode": "300250"
}
```
  </Step>
  
  <Step title="Test refund">
```bash
curl -X POST https://api-sandbox.nuvei.com/payment-api/payments/375011/refunds \
  -H "Content-Type: application/json" \
  -H "x-api-key: YOUR_SANDBOX_KEY" \
  -d '{
    "processingEntityId": "1234567890",
    "amount": 50,
    "currency": "USD"
  }'
```
  </Step>
</Steps>

## Sandbox Limitations

<Warning>
  Sandbox environment limitations:
  - No real money movement
  - Test cards only (real cards will fail)
  - Webhooks may have slight delays
  - Some edge cases may behave differently
</Warning>
