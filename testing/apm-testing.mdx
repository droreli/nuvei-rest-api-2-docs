# APM Testing Guide

<Warning>
APM test credentials only work in the integration environment. Ensure your merchantSiteId is configured for APM testing.
</Warning>

## PayPal

### Test Credentials

| Environment | Account Type | Credentials |
|-------------|--------------|-------------|
| Sandbox | Buyer | Email: `sb-buyer@business.example.com` |
| Sandbox | Buyer | Password: Contact Nuvei for test password |

### Test Flow

1. Select PayPal as payment method
2. Redirect to PayPal sandbox
3. Log in with test buyer credentials
4. Approve payment
5. Redirect back to merchant

### Test Scenarios

| Scenario | How to Trigger |
|----------|----------------|
| Approved | Complete normal flow with test buyer |
| Cancelled | Click "Cancel" on PayPal page |
| Pending | Use pending test account (contact Nuvei) |

## Apple Pay (Sandbox)

### Prerequisites

- Apple Developer account
- Sandbox Apple ID
- Safari on macOS/iOS

### Test Setup

1. Create sandbox tester in App Store Connect
2. Sign in to device with sandbox Apple ID
3. Add test card to Apple Wallet

1. Click Apple Pay button
2. Authenticate with Face ID/Touch ID (simulated in sandbox)
3. Transaction completes

## Google Pay (Sandbox)

### Test Setup

1. Join Google Pay test group
2. Use Chrome browser
3. Configure `environment: 'TEST'` in Google Pay API

### Test Configuration

```javascript
const paymentDataRequest = {
  environment: 'TEST',  // Use 'PRODUCTION' for live
  merchantInfo: {
    merchantId: 'YOUR_MERCHANT_ID',
    merchantName: 'Your Store Name'
  },
  // ... other configuration
};
```

### Test Flow

1. Click Google Pay button
2. Select test card from wallet
3. Authenticate
4. Transaction completes

## General APM Testing Tips

<AccordionGroup>
  <Accordion title="Handling Redirects" icon="arrow-right">
    - Test with `urlDetails.successUrl`, `failureUrl`, `pendingUrl`
    - Verify user returns to correct page based on outcome
    - Test browser back button behavior
  </Accordion>
  
  <Accordion title="Testing DMN (Webhooks)" icon="bell">
    - APMs often complete asynchronously via DMN
    - Configure DMN URL in Control Panel
    - Use webhook testing tools (ngrok, webhook.site)
    - Verify DMN signature validation
  </Accordion>
  
  <Accordion title="Currency Testing" icon="coins">
    - Test with APM's supported currencies
    - Some APMs only support specific currencies
    - Currency conversion may apply
  </Accordion>
  
  <Accordion title="Mobile Testing" icon="mobile">
    - Test APM flows on mobile browsers
    - Some APMs have mobile-specific flows
    - Test deep linking for app-based APMs
  </Accordion>
</AccordionGroup>

## APM Testing Checklist

<AccordionGroup>
  <Accordion title="Pre-Testing" icon="list-check">
    - [ ] APM enabled for merchantSiteId
    - [ ] Test credentials obtained
    - [ ] DMN URL configured
    - [ ] Return URLs configured
  </Accordion>
  
  <Accordion title="Happy Path Testing" icon="check">
    - [ ] Successful payment completes
    - [ ] User redirected to successUrl
    - [ ] DMN received with correct parameters
    - [ ] Transaction visible in Control Panel
  </Accordion>
  
  <Accordion title="Error Path Testing" icon="xmark">
    - [ ] User cancellation handled
    - [ ] Declined transactions handled
    - [ ] Timeout scenarios handled
    - [ ] Error messages displayed correctly
  </Accordion>
  
  <Accordion title="Edge Cases" icon="asterisk">
    - [ ] Browser back button
    - [ ] Page refresh during payment
    - [ ] Multiple tab/window handling
    - [ ] Session expiry handling
  </Accordion>
</AccordionGroup>
