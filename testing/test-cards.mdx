# Test Cards

Use these test cards in the **integration (sandbox) environment** to simulate various payment scenarios.

<Warning>
Test cards only work in the integration environment. No actual funds are transferred.
</Warning>

---

## General Rules

- CVV can be any 3-digit number (4 digits for Amex)
- Expiration date can be any future date
- Test cards won't work in production

### 3DS cardHolderName Patterns

Control 3DS flow using `cardHolderName` values:

<Tabs>
  <Tab title="Frictionless Browser">
    Use these values for frictionless authentication (no challenge):
    
    | cardHolderName | Description |
    |----------------|-------------|
    | `FL-BRW1` | Frictionless, fully authenticated |
    | `FL-BRW2` | Frictionless, attempts |
    | `FL-BRW3` | Frictionless, not authenticated |
    | `FL BRWA` | Frictionless with space |
    | `FL-BRW1-EXMPT` | Frictionless with exemption |
  </Tab>
  <Tab title="Challenge Browser">
    Use these values to trigger 3DS challenge:
    
    | cardHolderName | Description |
    |----------------|-------------|
    | `CL-BRW1` | Challenge, fully authenticated |
    | `CL-BRW2` | Challenge, attempts |
    | `CL-BRW3` | Challenge, not authenticated |
    | `CL BRWA` | Challenge with space |
  </Tab>
  <Tab title="Error Browser">
    Use these values to simulate 3DS errors:
    
    | cardHolderName | Description |
    |----------------|-------------|
    | `ERR-BRW1` | Error during authentication |
    | `ERR-BRW2` | Error with challenge |
    | `ERR-BRW3` | Protocol error |
  </Tab>
  <Tab title="Frictionless App">
    For in-app SDK testing:
    
    | cardHolderName | Description |
    |----------------|-------------|
    | `FL-APP1` | Frictionless, fully authenticated |
    | `FL-APP2` | Frictionless, attempts |

## Amount-Based 3DS Testing

Trigger specific 3DS flows using transaction amounts:

| Flow | Amount | Cards |
|------|--------|-------|
| **Frictionless** | `83.1` (any currency) | `5545060700001113`, `4567491000001113` |
| **Challenge** | `115.2` (any currency) | `5545060700002228`, `4567491000002228` |

<Tip>
Amount-based testing is useful when you can't control the cardholder name.
</Tip>

## Response Codes

### Successful Responses

| Status | transactionStatus | Description |
|--------|-------------------|-------------|
| Approved | `APPROVED` | Transaction successful |
| Pending | `PENDING` | Awaiting completion |

### Decline Responses

| Reason Code | Description |
|-------------|-------------|
| `-1` | General decline |
| `1001` | Insufficient funds |
| `1002` | Card expired |
| `1003` | CVV mismatch |
| `1004` | Invalid card number |
| `1005` | Do not honor |
| `1006` | Lost card |
| `1007` | Stolen card |
| `1008` | Card restricted |
