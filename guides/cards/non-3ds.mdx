# Non-3DS Card Payments

Process card payments without 3D Secure authentication—ideal for low-risk transactions.

<Warning>
  Non-3DS payments may have higher chargeback liability. Consider [3DS payments](/guides/cards/3ds-flow) for better fraud protection.
</Warning>

## When to Use

- Low-value transactions
- Trusted repeat customers
- Regions without 3DS mandates
- Quick checkout flows

## Request Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `processingEntityId` | string | ✓ | Your merchant ID |
| `transactionType` | string | ✓ | `Sale` or `Auth` |
| `amount` | number | ✓ | Amount in minor units |
| `currency` | string | ✓ | ISO 4217 currency code |
| `paymentMethod.type` | string | ✓ | Must be `card` |
| `paymentMethod.card.cardNumber` | string | ✓ | Full card number |
| `paymentMethod.card.expirationMonth` | string | ✓ | MM format |
| `paymentMethod.card.expirationYear` | string | ✓ | YYYY format |
| `paymentMethod.card.cvv` | string | ✓ | 3-4 digit CVV |
| `paymentMethod.card.cardHolderName` | string | | Cardholder name |

## Handling Declines

```json
{
  "paymentId": "375012",
  "result": {
    "status": "declined",
    "reason": {
      "code": "DECLINED",
      "message": "Insufficient funds"
    }
  }
}
```

**Common decline reasons:**
- Insufficient funds
- Invalid card number
- Expired card
- CVV mismatch
