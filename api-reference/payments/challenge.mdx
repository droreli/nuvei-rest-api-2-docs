# 3DS Challenge

Complete the 3DS flow after the customer has finished the challenge.

## Path Parameters

<ParamField path="paymentId" type="string" required>
  The payment ID from the initial payment request
</ParamField>

## Request Body

<ParamField body="processingEntityId" type="string" required>
  Your merchant processing entity ID
</ParamField>

## Example Request

<CodeGroup>
```bash cURL
curl -X POST https://api-sandbox.nuvei.com/payment-api/payments/375011/challenge \
  -H "Content-Type: application/json" \
  -H "x-api-key: YOUR_API_KEY" \
  -d '{
    "processingEntityId": "1234567890"
  }'
```

```javascript JavaScript
const response = await fetch(
  'https://api-sandbox.nuvei.com/payment-api/payments/375011/challenge',
  {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'x-api-key': 'YOUR_API_KEY'
    },
    body: JSON.stringify({
      processingEntityId: '1234567890'
    })
  }
);
```
</CodeGroup>

## Response

<ResponseField name="paymentId" type="string">
  Payment identifier
</ResponseField>

<ResponseField name="transactionId" type="string">
  Nuvei transaction ID
</ResponseField>

<ResponseField name="amount" type="number">
  Transaction amount
</ResponseField>

<ResponseField name="currency" type="string">
  Currency code
</ResponseField>

<ResponseField name="transactionType" type="string">
  Transaction type (Sale or Auth)
</ResponseField>

<ResponseField name="result" type="object">
  Final transaction result
  
  <Expandable title="result properties">
    <ResponseField name="status" type="string">
      `approved` or `declined`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="authCode" type="string">
  Authorization code (if approved)
</ResponseField>

<ResponseField name="threeD" type="object">
  3DS authentication details
  
  <Expandable title="threeD properties">
    <ResponseField name="authenticationStatus" type="string">
      Final 3DS status
    </ResponseField>
    <ResponseField name="flow" type="string">
      Will be `challenge`
    </ResponseField>
  </Expandable>
</ResponseField>

## 3DS Authentication Statuses

| Status | Description | Action |
|--------|-------------|--------|
| `Y` | Fully authenticated | Payment approved |
| `A` | Attempted | Payment may proceed |
| `N` | Not authenticated | Payment declined |
| `U` | Unable to authenticate | Merchant decision |
| `R` | Rejected | Payment declined |
