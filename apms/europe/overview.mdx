## Integration Example

### REST API - iDEAL Payment

```json
POST /ppp/api/v1/payment.do

{
  "sessionToken": "<sessionToken>",
  "merchantId": "<merchantId>",
  "merchantSiteId": "<merchantSiteId>",
  "clientRequestId": "<unique_request_id>",
  "clientUniqueId": "order_123",
  "amount": "99.00",
  "currency": "EUR",
  
  "paymentOption": {
    "alternativePaymentMethod": {
      "paymentMethod": "apmgw_iDeal",
      "BIC": "INGBNL2A"
    }
  },
  
  "billingAddress": {
    "firstName": "Jan",
    "lastName": "de Vries",
    "email": "jan@example.nl",
    "country": "NL"
  },
  
  "urlDetails": {
    "successUrl": "https://yoursite.com/success",
    "failureUrl": "https://yoursite.com/failure",
    "notificationUrl": "https://yoursite.com/webhooks"
  },
  
  "timeStamp": "<YYYYMMDDHHmmss>",
  "checksum": "<checksum>"
}
```

## Best Practices for Europe

<AccordionGroup>
  <Accordion title="Detect customer location" icon="location-dot">
    Use IP geolocation or billing address to show relevant APMs. Dutch customers expect iDEAL, Germans expect Sofort/Klarna.
  </Accordion>
  
  <Accordion title="Support multiple languages" icon="language">
    Use the `languageCode` parameter to show APM names in local language.
  </Accordion>
  
  <Accordion title="Consider BNPL in Germany" icon="clock">
    German shoppers strongly prefer "Buy Now, Pay Later" options. Consider Klarna or PayPal Pay Later.
  </Accordion>
  
  <Accordion title="Handle EUR and local currencies" icon="euro-sign">
    Most European APMs work with EUR, but Poland (PLN), Sweden (SEK), and others need local currency.
  </Accordion>
</AccordionGroup>
