# REST API 1.0 (Legacy)

REST API 1.0 is Nuvei's legacy server-to-server integration method.

<Warning>
  For new integrations, use [REST API 2.0](/rest-2/introduction) instead.
</Warning>

## Key Differences from REST 2.0

| Feature | REST 1.0 | REST 2.0 |
|---------|----------|----------|
| **URL Style** | Action-based (`/payment`) | Resource-based (`/payments`) |
| **Authentication** | Session + checksum | API key header |
| **Payment Lifecycle** | Multiple IDs | Single `paymentId` |
| **Operations** | `/refundTransaction` | `/payments/{id}/refunds` |

## Core Endpoints

- `/getSessionToken` → Session initiation
- `/payment` → Create payment
- `/settleTransaction` → Settle authorization
- `/refundTransaction` → Refund payment
- `/voidTransaction` → Void authorization
- `/getPaymentStatus` → Query payment status
